# NIST 800-171 Alignment Framework

Community-driven mapping of **NIST 800-171 Rev.2** controls to **Microsoft Security** products, Sentinel tables, and KQL queries.

> **[Browse the interactive reference](https://cyberlorians.github.io/nistframework/)** — filter by family, workload, or table and copy KQL directly into Sentinel.

---

## What This Is

A three-layer compliance solution for organizations pursuing NIST 800-171 Level 1:

| Layer | Purpose | Artifact |
|-------|---------|----------|
| **Reference** | Browse every control with its KQL queries, filter and search | [Interactive HTML](https://cyberlorians.github.io/nistframework/) |
| **Validation** | Prove your Sentinel workspace has the required tables and data | [ATO Workbook](ato/) — import into Sentinel |
| **Evidence** | Export compliance artifacts for auditors | CSV, Query Pack JSON |

### How it works

```
┌─────────────────┐    validate.py     ┌──────────────────┐

│                  │

│  NIST control    │──────────────────▶│  Alignment CSV    │
│                  │                   └──────────────────┘
│                  │    build_querypack ┌──────────────────┐
│                  │──────────────────▶│  Query Pack JSON  │
│                  │                   └──────────────────┘
│                  │    build_html.py   ┌──────────────────┐
│                  │──────────────────▶│  GitHub Pages     │
└─────────────────┘                    └──────────────────┘
```

**Contributors only touch YAML files** — everything else is auto-generated by CI on merge.

## Quick Start

### Browse controls
Visit **[cyberlorians.github.io/nistframework](https://cyberlorians.github.io/nistframework/)** — no install needed.

### Validate your environment
1. Open Microsoft Sentinel → Workbooks → Add workbook
2. Open Advanced Editor, paste the JSON from [`ato/ato_workbook.json`](ato/ato_workbook.json)
3. The workbook checks if each required log table exists and has data

### Import KQL queries into Sentinel
| Method | Steps |
|--------|-------|
| **Query Pack** | Deploy [`output/NIST_800-171_QueryPack.json`](output/NIST_800-171_QueryPack.json) as ARM template |
| **Watchlist** | Upload [`output/NIST_800-171_Alignment.csv`](output/NIST_800-171_Alignment.csv) as a Sentinel Watchlist |
| **Live reference** | `externaldata(...)[@"https://raw.githubusercontent.com/Cyberlorians/nistframework/main/output/NIST_800-171_Alignment.csv"]` |

## KQL Convention

Every query follows the **M-21-31 five-part structure**:

```kql
// Objective: What this query detects and why it matters for compliance
//
// ----- Part 0: Analyst-Driven Targeting (Optional) -----
// Leave arrays empty to return all results. Add specific values to scope.
let TargetUsers = dynamic([]);       // e.g., ["alice@contoso.com"]
//
// ----- Part 1: Base Filter -----
TableName
| where TimeGenerated > ago(30d)
//
// ----- Part 2: Parse & Extend -----
| extend ...
//
// ----- Part 3: Enrich -----
| extend Description = case(...)
//
// ----- Part 4: Apply Dynamic Filters -----
| where (array_length(TargetUsers) == 0 or UserPrincipalName in (TargetUsers))
//
// ----- Part 5: Final Output -----
| distinct ...
| sort by TimeGenerated desc
| take 50
```

**Key conventions:**
- `dynamic([])` with `array_length() == 0` — empty = all results, populated = scoped
- `SecurityEvent` queries include GPO path comments
- Every query ends with `| distinct` to deduplicate

## How to Contribute

See **[CONTRIBUTING.md](CONTRIBUTING.md)** for the full guide. The short version:

1. **Fork** this repo
2. **Copy** [`practices/_template.yaml`](practices/_template.yaml) to a new file
3. **Fill in** your control metadata and KQL queries
4. **Open a Pull Request** — CI validates automatically
5. **Maintainer reviews** and merges
6. **CI rebuilds** all artifacts and deploys to GitHub Pages

> You only edit YAML — the CSV, Query Pack, and HTML page are rebuilt automatically.
